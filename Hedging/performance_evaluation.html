Performance evaluation across models:¶

In my opinion, the best way of evaluating our model would be using out-sample multivariate normal log-likelihood.

The multivariate normal likelihood for a single observation is:

$$ \frac{1}{\sqrt{(2\pi)^d|\boldsymbol\Sigma|}} \exp\left(-\frac{1}{2}({x}-{\mu})^T{\boldsymbol\Sigma}^{-1}({x}-{\mu}) \right) $$

If we make our $\Sigma = \Sigma_t$ time-varying, taking logarithms and including n samples:

$$ l(\mu_t, \boldsymbol\Sigma_t) = -\frac{n}{2} log\left( (2 \pi)^d |\boldsymbol\Sigma_t| \right) - \frac{1}{2}\sum_{i=1}^{n} ({x_i}-{\mu_t})^T{\boldsymbol\Sigma_t}^{-1}({x_i}-{\mu_t}) $$

One simple way to evaluate a set of predictions would be to use one-day ahead predictions for the covariance matrix, i.e. at time t, give a prediction $\Sigma_{t+1}^p$ of the covariance matrix to be used for the next day. Also, it's important to remember that we are not trying to predict the means, so we can just use some sample fixed historical means or whatever other simplification we want ($\mu$).

Given this predictive setting, an evaluation interval and ignoring constants that don't depend in any parameter, we can just evaluate:

$$ likelihood = \sum_{t=1}^{n-1} \left[ -\frac{1}{2} log\left( |\boldsymbol\Sigma_{t+1}^p| \right) -\frac{1}{2} ({x_{t+1}}-{\mu})^T{\boldsymbol\Sigma_{t+1}^p}^{-1}({x_{t+1}}-{\mu}) \right]$$

Even though it might be a less intuitive measure than a mean square error, it makes much more sense since we can't observe the true instantaneous covariance matrix.

Please note that for the univariate case, assuming $\mu = 0$, the metric it's just the following:

$$ likelihood = \sum_{t=1}^{n-1} \left[ -\frac{1}{2} log\left( |\boldsymbol\sigma_{t+1}^2| \right) - \frac{1}{2} \frac{x_{t+1}^2}{\sigma_{t+1}^2 } \right]$$

The interpretation of this loss it's much more simple, the square value of return affects less the loss the larger it's the volatility (for the second term) whereas the first term penalizes innecessarily large volatilities for the model, this way a trade-off it's created so that the most likely model it's the one with a volatility large enough to account for the squared value of returns but not arbitrarily large.

Python implementation¶

We can make a very easy multivariate implementation of this metric, using tensors.

The most important part of the implementation it's the dimensionality of the input. We have n samples or timesteps and d dimensions for our multivariate Gaussian. Then:

  1. samples: it's a matrix of shape nxd
  2. mu: it's a matrix of shape nxd
  3. cov: it's a tensor of shape dxdxn

def perf_evaluation(samples, mu, cov, epsilon=1e-10):
    # Difference to the mean:
    d = samples - mu
    # We iterate over the n available time step. Each cov[:, :, i] it's a dxd covariance matrix that we need to invert. A small amount it's added to the diagonal in order to avoid
    # numerical problems with the inverse. Also, instead of the inverse, the moore-penrose inverse is used, in order to avoid problem with non-full rank covariance matrices.
    invcov = np.concatenate([np.linalg.pinv(cov[:, :, i] + epsilon*np.eye(cov.shape[0], cov.shape[0])).reshape(cov.shape[0], -1, 1) for i in range(cov.shape[2])], axis=2)
    # Once we have all the inverses, we can compute the sum of the n quadratic forms:
    temp = (np.sum(np.tile(d.reshape(mus.shape[0], 1, -1), (1, mus.shape[0], 1)) * invcov, axis=0))
    sumsquare = -0.5 * np.sum(temp*d, axis=0)
    # We need to take the log-determinant of the n covariance matrices:
    dets = np.array([np.linalg.slogdet(cov[:, :, i])[1] for i in range(cov.shape[2])])
    logdet = -0.5 * dets
    # We add up both contributions of the likelihood loss:
    loglike = sumsquare + logdet
    # We sum over all samples in order to return a single values, we can normalize with the number of samples:
    return np.sum(loglike)/(samples.shape[1]*samples.shape[0])

Synthetic:¶

In order to check if our metric it's actually a good measure of the goodness of our volatility estimator, we can generate several artificial returns, by sampling random samples from a distribution with zero mean an a specific time-varying volatility that will be used as ground truth.

In order to perform the experiment we are going to compare the performance at selecting as optimal the estimation with the least error (in this case we can compute the error because the volatility process it's been created by us!). As estimation methods, we are going to compare volatility estimation using rolling estimations with different rolling windows.

# Funcion to generate synthetic data with a specific characteristic window of variation for the process volatility.
def generate_synthetic(data, window):
    returns = data['EURUSD'].pct_change().dropna()
    ground_truth = returns.rolling(window).var().dropna()
    art_returns = pd.Series(index=ground_truth.index)
    for i in ground_truth.index:
        art_returns.loc[i] = np.sqrt(ground_truth.loc[i])*np.random.randn()
    return ground_truth, art_returns

# Evaluate the performances of all the estimates
def evaluate_performance(rets, forecasts):
    rr = rets.loc[forecasts.index]
    rr = rr.values.reshape(-1, 1).T
    mus = np.ones((1, rr.shape[1]))*np.mean(rr)
    # Performances:
    perfs = pd.Series(index=forecasts.columns)
    for col in forecasts:
        perfs.loc[col] = perf_evaluation(rr, mus, forecasts[col].values.reshape(1, 1, -1))
    return perfs

# Computing estimation error in our estimates:
def compute_errors(forecasts, ground_truth, rets):
    vol_error = (forecasts - np.tile(ground_truth.values.reshape(-1, 1), (1, forecasts.shape[1]))).abs().mean()
    pred_abs_val = np.sqrt(forecasts)*np.sqrt(2/np.pi)
    absval = np.tile(rets.abs().values.reshape(-1, 1), (1, forecasts.shape[1]))
    abs_error = (pred_abs_val - absval).abs().mean()
    return vol_error, abs_error

Conclusions:¶

  1. We propose to use multivariate normal loglikelihood in order to evaluate one-day ahead predictions of the covariance matrix.
  2. This outsample metric it's a well-defined and justified metric to compare all kind of models that generate predictions of the covariance matrix.
  3. The metric it's useful for the univariate and for the multivariate case and we essentially avoid choosing another estimation of the covariance in the next time step as ground truth for our models.

IMO we could use this metric and that way we could close all the pending issues related with covariance estimations and performance evaluation.
